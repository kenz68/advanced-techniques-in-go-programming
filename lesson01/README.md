# Lesson 01 — Course Introduction & Objectives

## Overview

This lesson sets the stage for the entire course. You will learn what topics are covered, who the course is designed for, and how the lessons are structured so you can get the most out of the material.

## Learning Objectives

By the end of this lesson you will be able to:

- Describe the scope of the course and the skills you will develop.
- Identify the prerequisite knowledge expected of course participants.
- Understand the practical, hands-on approach used throughout the course.
- Preview the four main technical areas: RESTful services, WebSocket protocols, generics, and utility development.

## Target Audience

| Background | Expectation |
|---|---|
| Intermediate Go developers | Familiar with Go basics; ready to learn idiomatic patterns and advanced APIs |
| Experienced developers from other languages | Comfortable with general programming; new to Go's concurrency model and standard library |

## Course Topics at a Glance

### RESTful Services
Build HTTP servers and clients using the Go standard library (`net/http`) and popular routers. Learn request routing, middleware, JSON serialisation, and authentication patterns.

### WebSocket Protocols
Implement real-time bidirectional communication using the WebSocket protocol. Explore connection upgrades, message framing, and broadcast patterns.

### Generics
Write type-safe, reusable data structures and algorithms using Go generics (type parameters). Understand constraints and when generics improve—or complicate—code.

### Practical Utility Development
Apply all of the above to build command-line tools and small services that solve real problems. Focus on clean project layout, error handling, and testing.

## How to Run This Lesson

```bash
cd lesson01
go run main.go
```

Expected output introduces the course and confirms your Go environment is ready.
